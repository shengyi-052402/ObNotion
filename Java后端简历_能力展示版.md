## 项目经历

### Life-Q&A 社区问答平台 | 后端开发 | 2026.04 - 2026.05

**项目简介：**  
一个面向生活问答与内容交流的社区平台，支持用户注册登录、帖子发布、评论回复、点赞收藏、消息通知、全文搜索、图片上传和后台管理等功能。项目采用前后端分离架构，后端围绕内容互动、搜索检索、异步通知和缓存加速等模块进行设计。

**技术栈：**  
`SpringBoot` `MyBatis-Plus` `MySQL` `Redis` `Kafka` `Elasticsearch` `JWT` `Spring Security` `OSS`

**项目亮点：**

- 负责用户登录与权限校验模块，基于 JWT 实现无状态认证，通过拦截器解析 Token 并将用户 ID 写入 ThreadLocal，方便业务层获取当前登录用户。
- 负责帖子详情缓存设计，针对详情页高频访问场景使用 Redis 缓存帖子主体、作者信息和互动数据，并引入随机 TTL 降低缓存集中失效风险。
- 负责点赞、收藏等互动模块，使用 Redis Set 维护用户互动状态，通过 Lua 脚本保证点赞/取消点赞操作的原子性，再结合定时或业务触发方式同步至 MySQL。
- 负责消息通知模块，将评论、回复、点赞等事件封装为 Kafka 消息，在事务提交后发布，避免业务回滚后仍产生脏通知。
- 设计通知消费幂等方案，消费端结合 Redis 去重 Key 控制重复消费，异常场景下回落到数据库记录，提升异步通知链路的可靠性。
- 负责搜索模块，使用 Elasticsearch 实现帖子标题、正文、标签等字段的全文检索，支持关键词高亮、搜索建议和数据库兜底查询。
- 负责搜索索引同步，将帖子新增、修改、删除事件通过 Kafka 异步同步到 Elasticsearch，降低主流程写入耗时，并保持搜索结果与业务数据基本一致。
- 接入 OSS 文件上传能力，支持用户上传帖子图片和头像，后端完成文件类型校验、路径生成和访问地址返回。

### 餐饮门店履约与运营管理系统 | 后端开发 | 

**项目简介：**  
一个面向餐饮门店的履约与运营管理系统，包含管理端和用户端。管理端支持员工、菜品、套餐、订单、门店状态和经营数据统计；用户端支持地址管理、购物车、下单、订单支付、催单和订单状态查看。项目重点围绕双端鉴权、订单状态流转、缓存优化、实时通知。

**技术栈：**  
`SpringBoot` `MyBatis` `MySQL` `Redis` `JWT` `WebSocket` `AOP`  `OSS` 

**项目亮点：**

- 负责管理端与用户端双端登录鉴权，分别使用 JWT 维护员工与用户身份，通过拦截器完成 Token 校验和登录态传递。
- 负责公共字段自动填充，基于 AOP 拦截新增、修改操作，统一填充创建人、创建时间、更新人和更新时间，减少重复样板代码。
- 负责菜品、套餐和门店状态缓存优化，将高频访问数据写入 Redis，并在新增、修改、停售等操作后主动删除相关缓存，保证查询性能和数据一致性。
- 负责订单核心流程，围绕待支付、待接单、制作中、派送中、已完成、已取消等状态设计订单状态流转，避免订单在异常路径下进入不可控状态。
- 负责订单超时处理，通过定时任务扫描待支付和派送中订单，自动取消超时未支付订单、自动完成长时间未确认订单。
- 负责催单和来单提醒能力，使用 WebSocket 建立管理端实时通信通道，将用户催单、支付成功等事件推送给门店端，减少人工刷新页面。
- 负责购物车和地址簿模块，支持用户维护收货地址、增删购物车商品、批量下单，并在下单时完成金额、地址、菜品状态等校验。
